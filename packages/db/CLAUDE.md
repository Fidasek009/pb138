<overview>
Shared database package. Both apps import types/DTOs from here; only the API connects to the database at runtime.
</overview>
<structure>
```
src/
├── schema/                 # Drizzle table definitions (pgTable, relations) — one file per domain
│   ├── user.ts
│   ├── post.ts
│   └── index.ts            # Barrel export of all tables and relations
├── client.ts               # Drizzle client setup, connection config
└── dto/                    # Zod schemas derived from Drizzle schema via drizzle-zod (createInsertSchema, createSelectSchema) — importable by frontend and backend
drizzle.config.ts           # Migration config
drizzle/                    # Generated migration SQL files (do not edit manually)
```
</structure>
<conventions>
- One schema file per domain entity.
- DTOs use `drizzle-zod` to derive Zod schemas from Drizzle table definitions — keeps DB schema and validation in sync.
- Migration files in `drizzle/` are generated by Drizzle Kit. Never edit them manually.
- Apps import the client from `client.ts`, not raw connection strings.
</conventions>
